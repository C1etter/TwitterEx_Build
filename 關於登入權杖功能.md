# 關於「導入/導出登入權杖」功能

## 關於此功能

它會導出當前登入帳戶的工作權杖。

此功能在以下情況下非常有用：

- 如果您已經在某個設備上登入，並且想要在另一個設備上登入
- 如果您需要重新安裝 TwitterEx
- 如果您想要登入克隆的 APK
- 如果您有一個已 root 的設備，並且能夠正常登入

權杖和其他必要數據將以 JSON 格式導出。  
您可以在登入界面使用它。

若要導入其他帳戶，請選擇「建立新帳戶」而非「加入現有帳戶」。

**重要提示：此功能並不能修復登入驗證問題。**

## 警告

導出的權杖屬於高度機密資訊，切勿與他人共享！  
如果他人獲得此權杖，他們可以自由訪問您的帳戶，直到您退出工作階段。

請注意您的剪貼簿或導出檔案。

## 工作原理

X for Android 會將工作權杖和一些快取信息存儲到 Android 系統的 AccountManager 中。  
即使您清除數據，這些信息仍然有效。  
（這也是為什麼即使清除應用程式數據，X 仍然保持登入狀態。）

因此，您可以在重新安裝後恢復這些信息並登入。

## 刪除帳戶時的注意事項

如果您在將權杖導入到另一個設備後，想要從某個設備中移除帳戶，請小心操作。

從設定中登出將登出與該權杖關聯的工作階段。  
因此，如果您僅按下設定中的登出按鈕，您也會被從使用相同權杖的其他設備中登出。

有兩種方法可以在不登出的情況下從應用程式中移除帳戶：

- 在飛行模式下按登出按鈕。（沒有互聯網連接）
- 如果應用程式當前只登入一個帳戶，可以直接卸載並重新安裝。

## 故障排除

**問：導入權杖後，立即出現「由於錯誤，您已被登出 @用戶名。」的通知。**

答：該工作階段已經登出，無法使用該權杖。

**問：導入權杖後，即使重新啟動應用程式，仍然未登入，且沒有通知。**

答：請允許 X 應用程式發送通知並重試。

## 附錄

### Piko 權杖檔案格式

```json
{
  "username": "",
  "token": "",
  "secret": "",
  "userdata": {
    "account_user_id": "",
    "account_state": "",
    "account_field_version": "",
    "account_user_type": "",
    "account_settings": "",
    "account_teams_contributor": "",
    "account_teams_contributees": "",
    "account_user_info": "",
    "account_can_access_x_payments": "",
    "account_is_x_payments_enrolled": "",
    "com.twitter.android.oauth.token.teamsContributeeUserId": ""
  }
}
```

- "token" 是來自 AccountManager 的認證權杖 `com.twitter.android.oauth.token` 的值
- "secret" 是來自 AccountManager 的認證權杖 `com.twitter.android.oauth.token.secret` 的值
- "userdata" 是存儲在 AccountManager 中的用戶資料，以鍵值對格式表示

並非所有 "userdata" 中的鍵會存在。
但某些鍵是必需的。如果導入帳戶時缺少任何必需的鍵，X 應用程式會在帳戶導入後立即移除該帳戶。

### 從已 root 設備獲取權杖（不使用 TwitterEx）

如果您在已 root 的設備上使用官方 X，而不是 TwitterEx，並且希望將權杖導出以便使用 TwitterEx，您可以通過以下方法提取權杖和用戶資料：

1. 使用任何支援 root 存取的檔案管理器應用程式
2. 取得 `/data/system_ce/0/accounts_ce.db`
3. 用任何 SQLite 資料庫瀏覽器/編輯器應用程式打開它
4. 打開 `accounts` 表
5. 記住您想要導出的帳戶的 `_id` 編號
6. 打開 `authtokens` 表
7. 查找 `accounts_id` 欄位為您記住的編號的紀錄
8. 將 `com.twitter.android.oauth.token` 和 `com.twitter.android.oauth.token.secret` 的值複製到上面 JSON 格式中的相應屬性
9. 打開 `extras` 表
10. 查找 `accounts_id` 欄位為您記住的編號的紀錄
11. 將所有鍵值對複製到上面 JSON 格式中的 `userdata` 屬性。請注意，`account_user_info` 本身是 JSON 文字，因此您必須轉義所有雙引號。
