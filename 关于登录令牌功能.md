# 关于“导入/导出登录令牌”功能

## 关于此功能

它会导出当前登录账号的会话令牌。

此功能在以下情况下非常有用：

- 如果你已经在某个设备上登录，并且想要在另一个设备上登录
- 如果你需要重新安装 TwitterEx
- 如果你想要登录克隆的 APK
- 如果你有一个已 root 的设备，并且能够正常登录

令牌和其他必要数据将以 JSON 格式导出。  
你可以在登录界面使用它。

若要导入其他账号，请选择“创建新账号”而非“添加已有账号”。

**重要提示：此功能并不能修复登录证明问题。**

## 警告

导出的令牌属于高度机密信息，切勿与他人共享！  
如果有人获得此令牌，他们则可以自由访问你的账号，直到你退出会话。

请注意你的剪贴板或导出文件。

## 工作原理

X for Android 会将会话令牌和一些缓存信息存储到 Android 系统的 AccountManager 中。  
即使你清除应用数据，这些信息仍然有效。  
（这也是为什么即使清除应用数据，X 仍然保持登录状态。）

因此，你可以在重新安装后恢复这些信息并登录。

## 删除账号时的注意事项

如果你在将令牌导入到另一个设备后，想要从某个设备中移除账号，请小心操作。

从设置中注销将注销与该令牌关联的会话。  
因此，如果你仅按下设置中的注销按钮，你也会被从使用相同令牌的其他设备中注销。

有两种方法可以在不注销的情况下从应用中移除账号：

- 在飞行模式下按注销按钮。（没有互联网连接）
- 如果当前只登录一个账号，可以直接卸载并重新安装。

## 故障排除

**问：导入令牌后，立即出现“由于错误，你已被注销 @用户名。”的通知。**

答：该会话已经注销，无法使用该令牌。

**问：导入令牌后，即使重新启动应用，也没有登录，并没有通知。**

答：请允许 X 应用发送通知，并重试。

## 附录

### Piko 令牌文件格式

```json
{
  "username": "",
  "token": "",
  "secret": "",
  "userdata": {
    "account_user_id": "",
    "account_state": "",
    "account_field_version": "",
    "account_user_type": "",
    "account_settings": "",
    "account_teams_contributor": "",
    "account_teams_contributees": "",
    "account_user_info": "",
    "account_can_access_x_payments": "",
    "account_is_x_payments_enrolled": "",
    "com.twitter.android.oauth.token.teamsContributeeUserId": ""
  }
}
```

- "token" 是来自 AccountManager 的验证令牌 `com.twitter.android.oauth.token` 的值
- "secret" 是来自 AccountManager 的验证令牌 `com.twitter.android.oauth.token.secret` 的值
- "userdata" 是存储在 AccountManager 中的用户数据，以键值对的格式表示

并非所有 "userdata" 中的键都会存在。
但某些键是必需的。如果导入账号时缺少任何必需的键，X 应用会在账号导入后立即移除该账号。

### 从已 root 设备获取令牌（不使用 TwitterEx）

如果你在已 root 的设备上使用官方 X，而不是 TwitterEx，并且希望将令牌导出以便使用 TwitterEx，你可以通过以下方法提取令牌和用户数据：

1. 使用任何支持 root 访问的文件管理器应用
2. 获取 `/data/system_ce/0/accounts_ce.db`
3. 用任何 SQLite 数据库浏览器/编辑器应用打开它
4. 打开 `accounts` 表
5. 记住你想要导出的账号的 `_id` 编号
6. 打开 `authtokens` 表
7. 查找 `accounts_id` 字段为你记住的编号的记录
8. 将 `com.twitter.android.oauth.token` 和 `com.twitter.android.oauth.token.secret` 的值复制到上面 JSON 格式中的相应属性
9. 打开 `extras` 表
10. 查找 `accounts_id` 字段为你记住的编号的记录
11. 将所有键值对复制到上面 JSON 格式中的 `userdata` 属性。请注意，`account_user_info` 本身是一个 JSON 文本，因此你必须转义所有双引号。
